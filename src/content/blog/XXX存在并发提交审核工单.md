# **XXX存在并发提交审核工单**

**1.漏洞详情**

**在对XXX进行安全测试时，发现其存在并发提交审核漏洞，能给工作人员造成严重工作负担**



**2.漏洞证明**

**创建设备，提交审核拦截抓包**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788683826-5f56d359-0f85-4fad-aabd-cd1e5a7c7377.png)**

**把 HTTP/2.0 改成 HTTP/1.1,并发，同一个工单交了多次**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788732310-c8436c5a-c7c3-490a-ae84-bed7d72224e8.png)**

**突破了限制，给审核造成大量工作负担**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788747840-615bc48a-7d0b-4880-b86f-a145a702eaab.png)**



**3.修复方案**

**用工单 ID + 用户 ID 生成 key，加 5 秒缓存锁（Redis / 内存都行）同一工单同一人，5 秒内只允许一次操作**
