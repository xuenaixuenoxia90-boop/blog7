# **XXX存在并发提交咨询单漏洞**

**1.漏洞详情**

**在对XXX进行安全测试时，发现其平台存在并发漏洞提交咨询单漏洞，突破一个验证码发一条咨询单限制，能给工作人员造成严重工作负担**



**2.漏洞证明**

**提交信息抓包**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788776462-2762b46d-4d81-4d1c-9011-b1cb3179da51.png)**

**利用 turbo 插件成功并发**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788793360-2f4d475b-7cdc-4a6c-8fc3-aed67f01dccd.png)**

**可以看到一条验证码只能提交一次咨询单，突破了限制**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788841115-0fed319d-4da4-4fb5-a4de-ff1d6df7d701.png)**

**![img](https://cdn.nlark.com/yuque/0/2026/png/60657001/1779788857249-08eb3781-615b-4a10-84c9-39e026b5ccd9.png)**

**3.修复方案**

**用工单 ID + 用户 ID 生成 key，加 5 秒缓存锁（Redis / 内存都行）同一工单同一人，5 秒内只允许一次操作**
